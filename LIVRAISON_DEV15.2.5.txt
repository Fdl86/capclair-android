Cause du plantage
-----------------
Le workflow ne plantait pas pendant la compilation Android ni pendant la génération du PDF.
Il échouait dès npm ci car 7 entrées resolved de package-lock.json pointaient vers un registre npm privé inaccessible depuis GitHub Actions.
Erreur observée : ETIMEDOUT sur packages.applied-caas-gateway1.internal.api.openai.org lors du téléchargement de pdf-lib 1.17.1.

Corrections
-----------
- Remplacement des 7 URL privées par les URL publiques registry.npmjs.org.
- Ajout de tools/check-lockfile-registry.mjs.
- Ajout de npm run lockfile:check.
- Contrôle du package-lock avant npm ci dans GitHub Actions.
- Trois tentatives de npm ci en cas de panne réseau transitoire.
- Incrément complet vers DEV15.2.5 et versionCode 15025.
- Artifact renommé cap-clair-dev15-2-5-debug-apk.
- Aucun changement du rendu PDF validé V5.
- Aucun changement du GPS, des traces, du Replay, du Suivi ou des exports GPX/JSON.
